## 🌍 Project Overview

This project transforms narrative historical information about queens and trade systems into:

- A structured historical dataset  
- An interactive governance dashboard  
- A comparative institutional modeling engine  
- A geospatial trade network visualization  

It demonstrates how governance systems can be structured, analyzed, and visualized computationally.

This is not simply a dashboard — it is a historical systems modeling prototype.

# 🔎 Core Analytical Modules

## 👑 Royal Council Filtering Engine
- Global full-text search  
- Maritime power classification  
- Empire filtering  
- Region filtering  
- Searchable queen selector  
- Story Mode vs Evidence Mode  

---

## 📜 Queen Governance Dossier
Each ruler is modeled as a structured system node including:

- Trade regions  
- Key ports  
- Major exports  
- Trade partners  
- Policy keywords  
- Economic impact summary  

---

## ⚖️ Structural Comparison Engine
Compare two rulers across:

- Shared trade regions  
- Shared ports  
- Shared exports  
- Unique governance domains  
- Maritime orientation  
- Reign duration  

This models institutional overlap rather than narrative similarity.

---

## 🧭 Governance Timeline (1474–1816)

Visualizes:

- Active rulers per year  
- Maritime-active rulers  
- Governance density over time  

This acts as a structural proxy for trade governance intensity.

# 🗺️ Geospatial Module (Folium)

Built with **Folium + Leaflet.js**.

### Features
- Toggle maritime vs land-oriented rulers  
- Toggle trade corridors  
- Vintage terrain & atlas-style base layers  
- Popup-based economic summaries  
- Layer control panel  

This extends the system from dashboard analytics into historical geospatial modeling.

# ▶️ Run Locally

Install dependencies:

```
pip install -r requirements.txt
```

Run Streamlit:

```
streamlit run app.py
```

Generate the trade map:

```
python map/interactive_trade_map.py
```

Open the generated map:

```
map/queens_trade_map.html
```
